# Битва Дашбордов

**Программный комплекс для анализа производительности персонала, занятого в продажах, на предприятиях под управлением CRM «МойСклад» и репрезентации статистики во внешних информационных системах.**

---

## О проекте

Проект разрабатывается в рамках **производственной практики** (12 недель, начиная с 11.02.2026). Цель — реализовать работающее приложение, которое:

- подключается к аккаунтам пользователей в облачной ERP **МойСклад**;
- собирает метрики продаж по сотрудникам (выручка, маржа и т.п.);
- формирует **ТОП-N** самых успешных продавцов за выбранный период;
- публикует одобренную сводку в **Telegram-канал** для мотивации команды.

---

## Целевая аудитория

- Владельцы магазинов и интернет-магазинов
- Менеджеры по продажам на маркетплейсах (Ozon, Wildberries, Яндекс Маркет и др.)
- Компании, ведущие учёт в **МойСклад** и заинтересованные в мотивации сотрудников

## Текущий стек

| Слой | Технологии |
|------|------------|
| **Фронтенд** | React 19, Vite, React Router v7, Axios |
| **Бэкенд** | Java 17, Spring Boot 3, Spring MVC, REST API |
| **БД** | PostgreSQL 15, Liquibase (миграции) |
| **Интеграции** | МойСклад JSON API, Telegram Bot API (пока моки) |
| **Инфраструктура** | Docker, Docker Compose, Nginx |
| **Тесты** | Mockito (unit), MockMvc (интеграционные, H2), Vitest + RTL (фронт) |

---

## Ключевые сущности и сценарии

- **Пользователь** — регистрация, привязка интеграций.
- **Интеграция с МойСклад** — токен для доступа к данным о продажах.
- **Интеграция с Telegram** — bot token + chat id канала для публикаций.
- **ТОП-N** — запрос → просмотр → подтверждение → публикация в Telegram → отмена.
- **Публикация** — факт отправки ТОП-N в Telegram с историей статусов.

## Запуск проекта

### Требования

- **Docker** и **Docker Compose**

### Запуск (рекомендуется)

```bash
git clone https://github.com/MolotilkaHolotilka/Dashboard-Battle.git
cd Dashboard-Battle
docker compose up --build
```

После запуска доступно:

| Сервис | URL |
|--------|-----|
| **Фронтенд** | http://localhost:3000 |
| **API (бэкенд)** | http://localhost:8080 |
| **Health check** | http://localhost:8080/actuator/health |

Демо-данные вставляются автоматически при старте:
- Пользователь: `demo@test.ru`
- Компания ID: `1`, с интеграциями МойСклад и Telegram

### Запуск бэкенда без Docker

```bash
# Требуется PostgreSQL на localhost:5432, БД dashboard_battle
cd backend
mvn spring-boot:run
```

### Запуск фронтенда без Docker

```bash
cd frontend
npm install
npm run dev   # http://localhost:5173
```
